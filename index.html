stepwise

Step-through tooling for things that didn't have step-through debuggers.

Three terminal-native TUIs. Shared design, narrow scope, zero hosted components.

The gap

Modern development produces artifacts that are hard to inspect step-by-step. Regex engines backtrack invisibly. Agent CLIs write thousands of JSONL entries per session. AI coding assistants edit dozens of files per turn. The existing tools either summarise after the fact (git commits, hosted dashboards, match highlights) or force a browser context-switch (regex101, Langfuse, LangSmith).

Stepwise is three narrow tools that cover this gap in the terminal — one per domain.

The tools

rgx

Step through regex execution — pattern editor, engine trace, backtrack heatmap.

  • Three engines (Rust regex, fancy-regex, PCRE2) with auto-selection based on pattern features
  • PCRE2 callout-based step-through debugger with dual-cursor and backtrack heatmap
  • Plain-English explanation generated from the regex AST
  • Code generation for 8 languages, stdin/pipe mode, test suite assertions, workspace save/load

cargo install rgx-cli · repo

agx

Step through agent sessions — user turns, tool calls, tool results on a navigable timeline.

  • Reads Claude Code, Codex CLI, and Gemini CLI session files directly — zero instrumentation
  • Bidirectional tool pairing: selecting a result shows the originating call input in one view
  • Time-travel scrub bar, filter, search, bookmarks, vim count prefixes, heatmap mode
  • --live watches a session as it's written; --diff compares two sessions; --summary prints a non-interactive digest

Not yet public — release coming soon.

sift

Step through AI-generated file writes — pre/post snapshots, accept or revert per change.

$ sift
session 2026-04-15T09:12 · 14 pending, 2 accepted

[pending] turn-3  src/auth/session.rs    +42 -8
[pending] turn-3  src/auth/mod.rs        +3 -0
[pending] turn-4  tests/auth_test.rs     +28 -0
[pending] turn-4  migrations/003.sql     +12 -0   (review: *.sql)

$ sift d turn-3
... unified diff ...
$ sift ok turn-3
$ sift undo turn-4
  • Hook-integrated with Claude Code, Gemini CLI, and Cline — intercepts every file Write/Edit/MultiEdit and Bash-tool mutation
  • Session-scoped ledger at .sift/ with pre-write snapshots; revert works on accepted entries too
  • Policy rules (.sift/policy.yml) to allow / review / deny paths; strict mode blocks next prompt until pending is cleared
  • sift sweep detects junk: exact duplicates, fuzzy duplicates, slop filenames, orphan markdown

Not yet public — release coming soon.

How they compose

Each tool is standalone. The utility compounds when they're used together on the same workflow.

1. Define the pattern — rgx

You need a policy rule in .sift/policy.yml that auto-allows edits to test files but reviews anything touching unsafe. Draft the regex in rgx with live match preview, step through with Ctrl+D to confirm the engine behaves as expected, copy the final pattern.

2. Run the agent

Claude Code / Codex / Gemini writes a JSONL session file as it works. Sift's hooks intercept each write and append to the .sift/ ledger with pre-snapshots.

3. Inspect the session — agx

Open the JSONL in agx. Scrub the timeline. See which user prompt led to which tool calls, which tool errored, where the assistant changed direction. Bookmark the turn where the writes you care about happened.

4. Review the writes — sift

Walk the ledger with sift review. Each entry shows the diff against the pre-snapshot. Accept, revert, or edit-then-accept. sift sweep flags junk files before you commit.

Shared principles

Terminal-native
No browser, no hosted dashboard, no web UI. Works over SSH, in containers, in air-gapped environments. Rust + ratatui + crossterm.
Zero instrumentation
Each tool reads artifacts that already exist. rgx reads patterns you type. agx reads the JSONL your agent CLI already writes. sift reads file state the OS already tracks. No SDK changes, no wrappers, no proxies.
Zero telemetry
Nothing is sent anywhere. Sessions, patterns, snapshots all stay on disk.
Cross-CLI
agx supports Claude Code, Codex CLI, and Gemini CLI out of the box. sift supports Claude Code, Gemini CLI, and Cline via hooks. New formats add as a parser module, not a rewrite.
Narrow scope, deep engineering
Each tool does one thing. rgx doesn't try to replace regex101's permalink sharing. agx doesn't try to replace Langfuse's team analytics. sift doesn't try to replace git. Where the scope ends, the tool stops.
Sub-grain
They operate below the layer where existing tools stop. Git works at commit grain; sift at sub-commit. Dashboards work at session grain; agx at step grain. Regex testers work at match grain; rgx at engine-step grain.

Install

cargo install rgx-cli

agx and sift are not yet public — install commands will land here when their repos flip public. Requires Rust 1.74+. Platform coverage: Linux, macOS, Windows (rgx); Linux, macOS (agx, sift).
